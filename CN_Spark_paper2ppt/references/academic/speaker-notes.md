# Speaker Notes · 演讲词写作规范

> ⚠️ **写入方式**：新 pipeline 下演讲词写入 `<project_path>/notes/total.md`，由 `scripts/total_md_split.py` 按页拆分，由 `scripts/svg_to_pptx.py --embed-notes`（默认开启）写入 PPTX 的备注区。`total.md` 格式：每页一节，节首 `# <NN>_<page_name>`（与 SVG 文件名一致），节间用 `---` 分隔。
>
> 本文件规定**写作内容**（长度、风格、节奏），对 SVG / python-pptx 两条路径通用。下文中提到的 `slide.notes_slide.notes_text_frame` 仅在 fallback 直写 PPTX 时相关。

下面是写法、长度、风格与各路线的具体要求。

## 总则

- **每页 100–180 字**（约 45–90 秒口播）；甘特图、概念框架页等核心页可放宽到 250 字。
- **不照读 PPT**。PPT 是骨架，演讲词是肌肉：补充背景、展开细节、给数字直觉解读、做章节衔接。
- **正式但自然**。不要书面到像念论文摘要，也不要口语化到像聊天。学术口语风格：用陈述句、少量第一人称（"我们""本文"），避免空洞的形容词。
- **重要数据要给"直觉"**。看到 `p = 0.001` 不要只读出来，要说"这意味着如果两组无差异，看到这个结果的概率不到千分之一"。
- **每页结尾留一句过渡**，自然引到下一页。最后一页例外，结尾应该是收束。

## 段落结构（推荐）

```
[1 句话点出本页主张]
[2–3 句展开 / 给证据 / 给直觉]
[1 句过渡到下一页]
```

## 中英文与术语

- 中文为主体，英文术语在第一次出现时给中文释义（"采用对比学习 contrastive learning，本质上是把相似样本拉近、不相似样本推远的目标函数"）；
- 数学符号用直白翻译："α 是学习率"，不要写"取 α 设定为 0.001"这种生硬翻译；
- 不要堆砌缩写。第一次出现 RAG 要写"检索增强生成（RAG）"，之后再用 RAG。

## 各路线的演讲词重点

### Route A · 学术论文

| 页 | 演讲词重点 |
|---|---|
| 封面 | 自报家门 + 用一句话说本文的最大新意（"这篇论文最有意思的一点是 X"） |
| 背景 | 为什么这个问题重要，跟听众有什么关系；不要照读 abstract |
| 方法 | 方法的**直觉**先于符号；"这个网络的核心思想是 X" |
| 结果 | 先说结论再说细节；统计量给直觉解读 |
| 局限 | 诚实点出，不要回避；这是答辩加分项 |
| 总结 | 强调"对我们组 / 对相关研究的启发是 X" |

### Route B · 课程报告

| 页 | 演讲词重点 |
|---|---|
| 背景 | 用近期一则新闻或一个具体场景把听众拉进议题 |
| 数据卡 | 每个数字解读"这意味着……"；不要只念数字 |
| 案例 | 先讲故事 / 时间线再给数据 |
| 观点 | 明确个人立场，课程报告允许有立场 |

### Route C · 开题报告

| 页 | 演讲词重点 |
|---|---|
| 背景 | 一句话说穿研究价值（"做成的话能解决 X 卡点"） |
| 国内外现状 | 按"国外做了什么 → 国内做了什么 → 都没做什么"递进 |
| 技术路线 | **边讲边指节点**；强调创新点出现在哪一步 |
| 甘特图 | 按里程碑讲（"前 4 个月做调研，6 月开始原型"），不要逐月念 |
| 创新点 | 明确"理论 / 方法 / 应用"分类；不超过 3 条 |
| 可行性 | 交代设备、数据、合作已落实；避免用"如果时间允许"这种弱化语 |

### Route D · 文献综述（重点路线）

| 页 | 演讲词重点 |
|---|---|
| 背景 | 解释**综述时机**："今年新出现 X，使得重新梳理领域变得必要" |
| 检索方法 | 给出数据库 + 关键词 + 入选数；要让评审能复现 |
| **概念框架页** | **这一页讲 3–5 分钟**：先点全图、再逐主题点过、最后强调主题间联系；这是综述讲解的高峰 |
| 主题详述 | 先点该主题在框架中的位置 → 方法谱系 → 1–2 个最强证据 |
| 主题间联系 / 争议 | 敢于点名争议双方，不要含糊用"有不同观点" |
| 研究空缺 | 每个空缺解释"为什么填这个空缺值得做"，跟未来方向连接 |
| 局限 | 综述本身的局限（语种、时间窗、数据库覆盖）也要交代 |

## 衔接句模板

学术演讲里几种好用的过渡：

- 上行：「在这一前提下，我们进一步看……」
- 转折：「然而，这一发现并没有完全解释……，因此……」
- 并列：「与之相对应的，另一类工作关注……」
- 对照：「相比之下，本文采用的策略是……」
- 收束：「综合上述三方面的证据，可以得出……」
- 综述特有：「这就引出了下一个主题——……」/「围绕这条主线还有另一条平行发展的路径——……」

## 反例（要避免）

❌ **照读 PPT**：
> "这一页讲的是研究背景。研究背景是 XXX。这就是研究背景。"

❌ **空洞形容词堆砌**：
> "本文具有非常重要的理论意义和深远的实践价值，对推动学科发展意义重大。"

❌ **念出符号但不解释**：
> "我们的目标函数是 L 等于负的 sigma 求和 y 乘以 log p。"

应改为：
> "目标函数是标准的交叉熵——简单来说就是惩罚模型把'对的类'判得不够自信。"

❌ **没有过渡的硬切**：
> "……以上是研究背景。下一页是研究方法。"

应改为：
> "了解了背景的紧迫性之后，我们看作者用什么方法来回应这个问题。"

## 与引文的协同

- 演讲词里引用他人时口播按"作者名 + 年份"（"Smith 2024 也观察到类似现象"），方便听众跟正文角标对上；
- 不要把整段 GB/T 7714 条目读出来；
- 引述别人观点时保持中性（"Smith 等人认为……"），不要带评判，除非这是综述/争议页。

## python-pptx 写入示例

```python
def add_speaker_note(slide, note_text: str):
    notes_tf = slide.notes_slide.notes_text_frame
    notes_tf.text = note_text  # 整段写入即可，notes 不要求富文本
```

`notes_text_frame.text` 直接赋值会覆盖所有 paragraph，最简单。如果要分段（推荐综述讲解的核心页分 3–4 段），用 `notes_tf.add_paragraph()`。
